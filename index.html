WISE Survey Units

Added an option for WISE units to be run as surveys that can be accessed without creating an account.

Technologies: Angular, TypeScript, Java Spring Boot, MySQL, Jasmine

Highlights
  • Built a REST API that automatically creates a temporary survey taker account, logs the user into that account, and then launches the unit.
  • Added support for creating and configuring survey runs within the authoring tool and generating shareable survey links.
  • Restricted survey takers to survey-related functionality, preventing access to areas of the platform intended for registered users.
  • Modified the database to support survey runs and survey taker accounts.

Context: Co-op project at the Technology Enhanced Learning in Science group at UC Berkeley.

Impact: Allows researchers to use WISE as a platform for creating and distributing surveys.

WISE Auto Translation

Added functionality to automatically translate text fields in WISE units using Amazon Translate.

Technologies: Amazon Translate, Angular, TypeScript, Java Spring Boot

Highlights
  • Added buttons to text fields in the authoring tool that automatically translate the text from the default language to the currently selected language.
  • Created a Spring Boot endpoint that receives translation requests from the front-end, passes them along to Amazon Translate, and then returns the translated text.
  • When a manually entered translation already exists, clicking the translate button opens a dialog that allows authors to choose between the existing translation and the machine translation.
  • Added support for translating HTML components while preserving the original structure and tags.

Context: Personal project completed while continuing to contribute to WISE in Spring 2026.

Impact: Enables unit authors to easily create multilingual WISE units.

Programming Language Implementation

Built a system in Java to run a Scheme-based programming language supporting interactions between typed and untyped modules.

Technologies: Java, JUnit

Highlights
  • Designed an Abstract Syntax Tree (AST) to capture all valid programs in the target language.
  • Built a parser that converts code in the target language into an AST.
  • Built validators that check for duplicate names, improperly formatted import statements, undeclared variables, and type errors.
  • Built the CESK state machine which takes a valid AST and produces an output or a runtime error, utilizing the Strategy pattern to cleanly implement its 20+ unique transition functions.
  • Implemented support for programs combining typed and untyped modules with cross-module imports, enforcing soundness through runtime proxies at module boundaries.
  • Wrote integration tests in the custom programming language and unit tests using JUnit.

Context: Semester project for CS4400 Programming Languages at Northeastern University. Completed with a partner.
